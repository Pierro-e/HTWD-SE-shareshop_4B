== Einführung
In diesem Dokument werden die wesentlichen Begriffe aus dem Anwendungsgebiet (Fachdomäne) der {project-system-name}'s definiert. Zur besseren Übersichtlichkeit sind Begriffe, Abkürzungen und Datendefinitionen gesondert aufgeführt.

== Begriffe

[%header]
|===
| Begriff | Definition und Erläuterung | Synonyme

| Task
| Tätigkeit, welche ein PBI (bzw. dessen Akzeptankriterium) teilweise realisiert
|

| Artikel
| Gegenstand, welcher eingekauft werden soll
| Item, Produkt

| Einkäufer
| WG-Mitglied, welcher gerade dabei ist Artikel einzukaufen
| -

| Schuldner
| Nutzer, der finanziell von andere Einkäufen profitiert
| -

| WG-Mitglied
| Nutzer der Software
| Bewohner, Mitglied


| Einkaufsliste
| Strukturierte Liste von Artikeln, die innerhalb einer WG gemeinsam beschafft werden sollen
| Besorgungslisten

| Produktinformationen
| Metadaten zu einem Produkt (z. B. Name, Beschreibung, Preis, Kategorie), die dem Benutzer helfen, eine Kauf- bzw. Auswahlentscheidung zu treffen
| Artikelinformationen

| Benutzer
| Mensch, der die Software nutzt
| Anwender, Nutzer

| Einkaufsprozess
| Abfolge definierter Schritte von der Produktauswahl über Einkauf bis zum zahlvorgang im System.
| Beschaffungsprozess

| Produktliste
| Tabelle oder Ansicht aller ausgewählten(zur Einkaufsliste hinzugefügten) Produkte, meist filter- und sortierbar
| Artikelliste

| Validierung
| Test, der prüft, ob ein Ergebnis den spezifizierten Anforderungen und dem Nutzerbedarf entspricht
| Überprüfung, Verifizierung

| Favoriten-Items
| Artikel, die der Bewohner öfter kaufen möchte und deshalb in einer persönlichen extra Liste für schnelleren zugriff Gespeichert hat.  
| Favoriten
|===

== Verzeichnis der Datenstrukturen

[%header]
|===
| Bezeichnung | Definition | Format | Gültigkeitsregeln | Aliase

| Nutzer
| Registrierter Benutzer der Anwendung mit Login- und Profildaten.
| `id` INT +
`email` VARCHAR(255) +
`name` VARCHAR(150) +
`passwort_hash` VARCHAR(255) +
`theme` INT +
`color` INT +
`DecayDays` DECIMAL(10,2)
|  `id` ist Primärschlüssel (PK) und eindeutig  +
 `email` muss eindeutig sein und sowohl `@` als auch `.` enthalten  +
 `name` not null  +
 `passwort_hash` not null  +
 `theme` ist optional (nullable), Default = 0  +
 `color` ist optional (nullable), Default = 0  +
 `DecayDays` ist optional (nullable), Default = 7.00
| User

| Listen
| Einkaufsliste, die von einem Nutzer angelegt wird.
| `id` INT +
`name` VARCHAR(150) +
`ersteller` INT +
`datum` DATE
|  `id` PK, eindeutig  +
 `name` darf nicht leer sein  +
 `ersteller` ist optional (nullable); falls gesetzt: FK -> `Nutzer.id`  +
+  `datum` ist optional (nullable); falls gesetzt: gültiges Datum

| ShoppingList

| ListenMitglieder
| Verknüpfung, welche Nutzer Teil welcher Liste sind.
| `listen_id` INT +
`nutzer_id` INT +
`beigetreten_am` DATE
|  Zusammengesetzter PK (`listen_id`,`nutzer_id`)  +
 `listen_id` FK → `Listen.id`  +
 `nutzer_id` FK → `Nutzer.id`
| ListMembers

| ListeProdukte
| Produkte, die zu einer Liste gehören, inkl. Menge & Beschreibung.
| `listen_id` INT +
`produkt_id` INT +
`produkt_menge` DECIMAL(10,2) +
`einheit_id` INT +
`hinzugefügt_von` INT +
`beschreibung` TEXT
|  Zusammengsetzter PK (`listen_id`,`produkt_id`,`hinzugefügt_von`)  +
 `listen_id` FK -> `Listen.id`  +
 `produkt_id` FK -> `Produkt.id`  +
 `einheit_id` ist optional (nullable); falls gesetzt: FK -> `Einheiten.id`  +
 `hinzugefugt_von` FK -> `Nutzer.id`  +
 `produkt_menge` ist optional (nullable); falls gesetzt: Mengenwert ≥ 0
| ListItems

| Einkaufsarchiv
| Abgeschlossener Einkauf einer Liste (Historie).
| `einkauf_id` INT +
`listen_id` INT +
`eingekauft_von` INT +
`eingekauft_am` DATE +
`gesamtpreis` DECIMAL(10,2)
|  `einkauf_id`, eindeutig, auto increment PK  +
 `listen_id` not null; FK → `Listen.id`  +
 `eingekauft_von` ist optional (nullable); falls gesetzt: FK → `Nutzer.id`  +
 `eingekauft_am` ist optional (nullable); falls gesetzt: gültiges Datum +
 `gesamtpreis` ist optional (nullable); falls gesetzt:`gesamtpreis` ≥ 0
| Purchase

| eingekaufte_Produkte
| Produkte, die in einem bestimmten Einkauf gekauft wurden.
| `id` INT +
`einkauf_id` INT +
`produkt_id` INT +
`produkt_menge` DECIMAL(10,2) +
`einheit_id` INT, +
`produkt_preis` DECIMAL(10,2) +
`hinzugefugt_von` INT +
`beschreibung` TEXT
| `id` PK, eindeutig, auto increment +
 `einkauf_id` not null; FK → `Einkaufsarchiv.einkauf_id`  +
 `produkt_id` not null; FK → `Produkt.id`  +
 `einheit_id` ist optional(nullable); falls gesetzt: FK → `Einheiten.id`  +
 `produkt_menge` ist optional (nullable); falls gesetzt: Menge ≥ 0  +
 `produkt_preis` ist optional (nullable); falls gesetzt: Preis ≥ 0  +
 `hinzugefuegt_von` ist optional (nullable); falls gesetzt: FK → `Nutzer.id`  +
 `beschreibung` ist optional (nullable)
| PurchasedItems

| Produkt
| Katalogeintrag eines verfügbaren Produkts.
| `id` INT +
`name` VARCHAR(255)
|  `id` PK  +
 `name` unique, not null +
| Item

| Einheiten
| Maßeinheiten für Mengenangaben.
| `id` INT +
`name` VARCHAR(20) +
`abkurzung` VARCHAR(10)
|  `id` PK  +
 `name` unique, not null +
 `abkurzung` ist optional (nullable)
| Units

| Bedarfsvorhersage
| Speichert nutzerspezifische Zähler/Informationen zur Bedarfsvorhersage je Produkt.
| `nutzer_id` INT +
`produkt_id` INT +
`counter` DECIMAL(10,2) +
`last_bought` TIMESTAMP
|  Zusammengesetzter PK (`nutzer_id`,`produkt_id`)  +
`nutzer_id` FK → `Nutzer.id`  +
`produkt_id` FK → `Produkt.id`  +
`counter` ist optional (nullable), Default = 0.00; falls gesetzt: counter ≥ 0  +
`last_bought` ist optional (nullable), Default = CURRENT_TIMESTAMP
| DemandForecast


| fav_Produkte
| Favorisierte/gespeicherte Produkte eines Nutzers (z.B. als Vorlage für Listen).
| `nutzer_id` INT +
`produkt_id` INT +
`menge` DECIMAL(10,2) +
`einheit_id` INT +
`beschreibung` TEXT
|  Zusammengesetzter PK (`nutzer_id`,`produkt_id`)  +
`nutzer_id` FK → `Nutzer.id`  +
`produkt_id` FK → `Produkt.id`  +
`einheit_id` ist optional (nullable); falls gesetzt: FK → `Einheiten.id`  +
`menge` ist optional (nullable); falls gesetzt: menge ≥ 0
| FavoriteItems

| Kostenaufteilung
| Speichert, wer wem wie viel Geld schuldet (Empfänger ↔ Schuldner).
| `empfaenger_id` INT +
`schuldner_id` INT +
`betrag` DECIMAL(10,2)
|  Zusammengesetzter PK (`empfaenger_id`,`schuldner_id`)  +
`empfaenger_id` FK → `Nutzer.id`  +
`schuldner_id` FK → `Nutzer.id`  +
`betrag` ist optional (nullable); falls gesetzt: betrag ≥ 0
| CostSplit


|===
