== Einführung
In diesem Dokument werden die wesentlichen Begriffe aus dem Anwendungsgebiet (Fachdomäne) der <Projektthema-Systems> definiert. Zur besseren Übersichtlichkeit sind Begriffe, Abkürzungen und Datendefinitionen gesondert aufgeführt.

== Begriffe

[%header]
|===
| Begriff | Definition und Erläuterung | Synonyme

| *Task*
| Tätigkeit, welche ein PBI (bzw. dessen Akzeptankriterium) teilweise realisiert
|

| *Artikel*
| Gegenstand, welcher eingekauft werden soll
| -

| Einkäufer
| WG-Mitglied, welcher gerade dabei ist Artikel einzukaufen
| -

| Schuldner
| Nutzer, der finanziell von andere Einkäufen profitiert
| -

| WG-Mitglied
| Nutzer der Software
| -


| Einkaufslisten
| Strukturierte Liste von Artikeln, die innerhalb einer WG gemeinsam beschafft werden sollen
| Besorgungslisten

| Funktionalität
| Gesamtheit aller vom System bereitgestellten Funktionen, die den Nutzern messbaren Nutzen geben und fachliche Anforderungen realisieren
| Leistungsumfang

| Komponenten
| Abgegrenzte Bausteine des Softwaresystems mit klarer Verantwortung und definierten Schnittstellen; können unabhängig entwickelt, getestet und ausgetauscht werden
| Bauteile, Module

| Produktinformationen
| Metadaten zu einem Produkt (z. B. Name, Beschreibung, Preis, Kategorie), die dem Benutzer helfen, eine Kauf- bzw. Auswahlentscheidung zu treffen
| Artikelinformationen

| Benutzer
| Mensch, der die Software nutzt
| Anwender, Nutzer

| Datenbank
| Verwaltungssystem zur Speicherung, Abfrage und Konsistenzsicherung von Daten für eine Anwendung
| 

| Datenmodell
| Abstrakte Beschreibung der relevanten Entitäten und ihrer Beziehungen; legt fest, wie Informationen strukturiert und in der Datenbank abgespeichert werden
| Informationsmodell

| Datenstruktur
| Interne Organisation von Daten in Speicher oder Dateien (z. B. Arrays, Hash-Maps)
| Datenschema

| Einkaufsprozess
| Abfolge definierter Schritte von der Produktauswahl über Einkauf bis zum zahlvorgang im System.
| Beschaffungsprozess

| Funktion
| Einzelne, klar umrissene Aufgabe oder Prozedur innerhalb des Systems, die einen konkreten Output erzeugt
| Aufgabe

| Schnittstelle
| Formal definierter Zugriffspunkt, über den Komponenten oder externe Systeme miteinander kommunizieren
| 

| Komponentenübersicht
| Zusammenstellung aller Systemkomponenten inklusive ihrer Verantwortlichkeiten und Beziehungen -> liefert einen Architektur-überblick
| Komponentenliste

| Produktliste
| Tabelle oder Ansicht aller ausgewählten(zur Einkaufsliste hinzugefügten) Produkte, meist filter- und sortierbar
| Artikelliste

| Qualitätsmerkmal
| Eigenschaft, anhand der sich die Qualität eines Produkts oder Systems beurteilen lässt
| Qualitätskriterium

| Systemanforderung
| Funktionale oder nicht-funktionale Bedingung, die das Softwaresystem erfüllen muss, um Stakeholder-Bedürfnisse zu erfüllen
| Systemvoraussetzung

| Systemkomponente
| Abgeschlossene Einheit innerhalb des Gesamtsystems, die einen Teil der Infrastruktur bereitstellt
| 

| Validierung
| Test, der prüft, ob ein Ergebnis den spezifizierten Anforderungen und dem Nutzerbedarf entspricht
| Überprüfung, Verifizierung
|===

== Verzeichnis der Datenstrukturen

[%header]
|===
| Bezeichnung | Definition | Format | Gültigkeitsregeln | Aliase

| Nutzer
| Registrierter Benutzer der Anwendung mit Login- und Profildaten.
| `id` INT, `email` VARCHAR(255) +
`name` VARCHAR(150) +
`passwort_hash` VARCHAR(255)
|  `id` ist Primärschlüssel (PK) und eindeutig  +
 `email` muss eindeutig sein und sowohl `@` als auch `.` enthalten  +
 alle Felder not null
| User

| Listen
| Einkaufsliste, die von einem Nutzer angelegt wird.
| `id` INT +
`name` VARCHAR(150) +
`ersteller` INT +
`datum` DATE
|  `id` PK, eindeutig  +
 `name` darf nicht leer sein  +
 `ersteller` referenziert `Nutzer.id`  +
 `datum` muss gültiges Datum sein
| ShoppingList

| ListenMitglieder
| Verknüpfung, welche Nutzer Teil welcher Liste sind.
| `listen_id` INT +
`nutzer_id` INT +
`beigetreten_am` DATE
|  Zusammengesetzter PK (`listen_id`,`nutzer_id`)  +
 `listen_id` FK → `Listen.id`  +
 `nutzer_id` FK → `Nutzer.id`
| ListMembers

| ListeProdukte
| Produkte, die zu einer Liste gehören, inkl. Menge & Beschreibung.
| `listen_id` INT +
`produkt_id` INT +
`produkt_menge` DECIMAL(10,2) +
`einheit_id` INT +
`hinzugefugt_von` INT +
`beschreibung` TEXT
|  PK (`listen_id`,`produkt_id`)  +
 `einheit_id` FK → `Einheiten.id`  +
 `hinzugefugt_von` FK → `Nutzer.id`  +
 Mengenwert ≥ 0
| ListItems

| Einkaufsarchiv
| Abgeschlossener Einkauf einer Liste (Historie).
| `id` INT + `listen_id` INT + 
`getatigt_von` INT +
`getatigt_am` DATE + 
`gesamtpreis` DECIMAL(10,2)
|  `id` PK  +
 `listen_id` FK → `Listen.id`  +
 `getatigt_von` FK → `Nutzer.id`  +
 `gesamtpreis` ≥ 0 +
| Purchase

| eingekaufte_Produkte
| Produkte, die in einem bestimmten Einkauf gekauft wurden.
| `einkauf_id` INT +
`produkt_id` INT + 
`produkt_menge` DECIMAL(10,2) +
`einheit_id` INT, `produkt_preis` DECIMAL(10,2) +
`hinzugefugt_von` INT
| PK (`einkauf_id`,`produkt_id`)  +
 `einkauf_id` FK → `Einkaufsarchiv.id`  +
 `produkt_id` FK → `Produkt.id`  +
 `einheit_id` FK → `Einheiten.id`  +
 Preis & Menge ≥ 0 +
| PurchasedItems

| Produkt
| Katalogeintrag eines verfügbaren Produkts.
| `id` INT +
`name` VARCHAR(255)
|  `id` PK  +
 `name` unique, not null +
| Item

| Einheiten
| Maßeinheiten für Mengenangaben.
| `id` INT +
`name` VARCHAR(20) +
`abkurzung` VARCHAR(10)
|  `id` PK  +
 `name` unique, not null +
 `abkurzung` not null +
| Units
|===
