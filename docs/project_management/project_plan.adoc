== Praktiken und Bewertung
* wöchentliches *Online Meeting* - Freitag 12:00 bis 13:30

=== Definition of Ready - Product Backlog Item

* Product Backlog Item muss vom Product Owner angenommen werden -> kein Label "vorschlag"
* Field "Status" ist auf "ready" gesetzt
* Akzeptankriterien sind ausgearbeitet und prüfbar

=== Definition of Done - Product Backlog Item

* Alle Sub-Issues (Tasks und Akzeptanzkriterien) sind abgeschlossen.
* Der Code wurde reviewed und abgenommen.

== Deployment

* aktuellen Zustand im wöchentlichen Meeting besprechen
* bei Bedarf Remote "Developer" Branch in Remote "Main" Branch mergen

== Erkenntnisse (Lessons learned)

=== Version Control System Workflow

Version Control Software unterstützt und ermöglicht unabhängige bis zu "chaotischen" Arbeitsweisen.
Dennoch ist habe wir festgestellt, dass es hilfreich sein kann eine Workflow zu defninieren, welcher zusätzlichen Arbeitsaufwand minimiert.
